## Design Context

### Users
End users — not developers. University graduation project evaluators, peers, and anyone curious about these three AI tools. They access the hub from a browser on desktop/laptop, likely in a demo or presentation context. The job: quickly understand what each project does and launch it.

### Brand Personality
Refined. Purposeful. Confident.
Not a startup landing page. Not a developer dashboard. A curated product foyer — three tools presented as a cohesive family, each with its own character.

### Aesthetic Direction
- Dark theme (demo/presentation context, warm ambient tone)
- Editorial, magazine-like information hierarchy — not a card grid
- Warm OKLCH neutrals tinted toward brand hue (~55)
- Restrained accent colors: amber (ai-spiker), sage (mac_chatbot), crimson (var)
- Typography-driven hierarchy over decorative elements
- Generous whitespace, asymmetric where useful
- Subtle ambient mesh background (very low chroma, slow drift)
- Spring physics for key interactions only (not everywhere)

### Anti-references
- Generic SaaS dashboards, shadcn demo pages
- AI landing pages with neon glow, glassmorphism, gradient text
- Three identical cards in a row
- Excessive borders, dividers, card nesting
- Monospace-as-personality

### Design Principles
1. **Every element earns its place.** If it doesn't serve hierarchy, action, or identity — remove it.
2. **Typography is the design.** Size, weight, and space create hierarchy. Not boxes, not shadows.
3. **Color is currency — spend it wisely.** 60% warm neutrals, 30% secondary text, 10% project accents.
4. **Motion is purposeful.** One spring-physics hover on the CTA. One fade-up on load. Nothing gratuitous.
5. **Editorial, not template.** Numbered sections, varied rhythm, confident whitespace.
